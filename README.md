# 微电子学习问答助手项目报告

## ① 题目、摘要与关键词

**题目：** 微电子学习问答助手

**摘要：** 本项目面向微电子、半导体器件、CMOS 电路和集成电路基础课程学习场景，构建了一个基于 DeepSeek 大语言模型 API 的学习问答网页。系统通过 Prompt 工程限定模型回答范围，使其聚焦课程相关知识，并支持用户在网页中输入自己的 DeepSeek API Key 后直接调用模型。前端采用 HTML、CSS 和 JavaScript 实现，部署在 GitHub Pages 上，具备示例问题、对话式问答、Markdown 渲染、回答复制、空输入和 API Key 错误提示等功能。

**关键词：** 微电子；半导体器件；DeepSeek；大语言模型；Prompt 工程；GitHub Pages；学习问答系统

## ② 项目背景与问题分析

微电子课程通常包含半导体物理、PN 结、MOSFET、CMOS 反相器、短沟道效应、版图规则和工艺流程等内容。这些知识同时涉及物理、电路、材料、数学建模和工程实现，初学者容易出现以下问题：

- 概念抽象，例如能带、内建电场、阈值电压等难以直观理解。
- 知识跨度大，物理机理、电路表现和工艺限制之间的联系不明显。
- 学习资料分散，学生需要在教材、课件和网络资料之间反复查找。
- 通用聊天机器人容易回答范围过宽，不一定适合课程学习。

因此，本项目选择“微电子学习问答助手”作为明确应用场景，目标不是做普通聊天工具，而是通过限定 Prompt 和前端交互设计，帮助学生围绕课程概念进行提问、解释和复习。项目重点解决“课程概念解释不够直观”“问题需要跨学科视角理解”“缺少轻量化随问随答工具”这几类学习痛点。

## ③ 模型选择与技术路线

### 模型选择

本项目使用 DeepSeek 云端 API，默认模型为：

```text
deepseek-v4-flash
```

选择理由：

- DeepSeek 是公开可获得的大语言模型服务，接口文档清晰，便于复现实验流程。
- 云端 API 使用门槛低，不需要本地显卡或复杂模型部署环境。
- 对中文问答和课程概念解释较友好，适合教育辅助类应用。
- 成本相对较低，适合课程项目演示和个人学习。

### 技术路线

项目采用“静态网页 + 浏览器直连 DeepSeek API + Prompt 工程”的路线：

```text
用户输入 API Key 和问题
        ↓
前端 JavaScript 读取输入
        ↓
加入微电子限定系统 Prompt
        ↓
调用 DeepSeek Chat API
        ↓
返回模型回答
        ↓
网页以对话气泡和 Markdown 格式展示
```

本项目未进行模型微调、LoRA/QLoRA 训练或 RAG 知识库检索。这样设计的原因是项目目标是完成一个轻量、可部署、可演示的课程学习助手，优先保证流程完整、界面清晰和模型调用真实可用。

## ④ 系统设计与功能实现

### 系统架构

系统由静态前端页面组成，主要文件如下：

```text
public/
├── index.html   页面结构
├── styles.css   页面样式
├── script.js    交互逻辑、Prompt、API 调用、Markdown 渲染
└── CNAME        GitHub Pages 自定义域名配置
```

部署方式：GitHub Pages。

线上地址：

```text
https://ai-for-me.xiaocaiziyou.top/
```

### 功能模块

- **API Key 输入模块：** 用户自行填写 DeepSeek API Key。Key 仅用于当前浏览器会话，不写入仓库，也不默认保存到本地存储。
- **问题输入模块：** 用户输入微电子学习问题，页面提供 MOSFET、CMOS、PN 结、短沟道效应、DRC/LVS 等示例问题。
- **模型调用模块：** 前端使用 `fetch` 直接请求 DeepSeek Chat API。
- **Prompt 约束模块：** 系统 Prompt 限定回答范围，要求模型聚焦微电子课程内容。
- **对话展示模块：** 用户问题显示在右侧，助手回答显示在左侧，形成对话式学习体验。
- **Markdown 渲染模块：** 支持标题、列表、引用、代码块、行内代码、链接等基础 Markdown 格式。
- **复制模块：** 助手成功回答后提供复制按钮，方便学生保存学习笔记。
- **异常提示模块：** 对空问题、未填写 API Key、API Key 无效、网络异常等情况给出提示。

### 核心流程

1. 用户打开网页。
2. 用户填写 DeepSeek API Key。
3. 用户输入或点击示例问题。
4. 前端检查 API Key 和问题是否为空。
5. 页面追加右侧用户消息气泡。
6. 页面显示“正在思考，请稍候。”状态。
7. 前端携带系统 Prompt 和用户问题调用 DeepSeek API。
8. 页面追加左侧助手回答气泡。
9. 用户可点击复制按钮复制回答内容。

## ⑤ Prompt

本项目通过系统 Prompt 限定模型角色、回答范围和回答风格。核心设计包括：

- **角色设定：** 模型被设定为“微电子学习问答助手”，服务对象是学习微电子、集成电路与半导体基础课程的学生。
- **范围限制：** 只回答半导体物理、半导体器件、CMOS 电路、集成电路基础、版图与工艺、数字/模拟电路中与微电子课程相关的问题。
- **回答风格：** 使用中文，结构清晰，适合课程学习；优先解释概念、公式含义、直观理解和常见误区。
- **特色要求：** 如果可以，从不同学科的视角解释问题，帮助用户建立跨学科理解。
- **拒答策略：** 对旅游、电影、诗歌等明显超出微电子范围的问题礼貌拒绝，并引导用户回到微电子学习主题。
- **可靠性约束：** 不编造教材页码、论文、实验数据或不存在的课程资料。

当前项目没有使用外部工具调用、智能体工具链或 RAG 检索。模型使用方式为 DeepSeek Chat API 调用和 Prompt 工程。

## ⑥ 实验测试与效果分析

### 测试环境

- 前端部署：GitHub Pages
- 浏览器：桌面浏览器
- 模型：DeepSeek API，默认 `deepseek-v4-flash`
- 调用方式：用户在网页中填写自己的 DeepSeek API Key 后发起请求

### 代表性测试结果

| 测试类型 | 代表性输入 | 模型输出摘要 | 人工评价 |
| --- | --- | --- | --- |
| 基础概念 | 请解释 PN 结的内建电场。 | 解释 P 区和 N 区接触后载流子扩散、空间电荷区形成、内建电场方向及其对扩散的抑制作用。 | 回答符合课程知识，能帮助理解 PN 结平衡状态。 |
| 电路问题 | CMOS 反相器为什么静态功耗低？ | 说明稳定状态下 PMOS 或 NMOS 只有一个导通，理想情况下电源到地没有直流通路，主要功耗来自动态翻转和漏电。 | 回答准确，能区分静态功耗、动态功耗和漏电。 |
| 器件问题 | MOSFET 的阈值电压受哪些因素影响？ | 从掺杂浓度、氧化层厚度、栅材料功函数、体效应、短沟道效应、工艺偏差等方面解释。 | 覆盖因素较全，适合作为复习提纲。 |
| 较难问题 | 短沟道效应为什么会影响阈值电压？ | 解释沟道变短后源漏耗尽区影响增强，栅控能力下降，DIBL 等效应导致阈值电压变化。 | 能从器件结构和电场控制角度说明原因。 |
| 范围外问题 | 帮我写一篇旅游攻略。 | 礼貌拒绝普通旅游攻略请求，并提示可以提问微电子、半导体器件或集成电路相关问题。 | 符合 Prompt 限定，能避免变成通用聊天机器人。 |
| 异常输入 | 未填写 API Key 直接提交 | 页面提示“请输入你的 DeepSeek API Key 后再提交。” | 前端校验有效，避免无意义 API 请求。 |
| 异常输入 | 使用无效 API Key 提问 | 页面显示 DeepSeek API 鉴权失败信息。 | 错误提示清晰，便于用户定位问题。 |

### 效果分析

测试表明，系统能够完成输入、Prompt 处理、模型调用、输出展示和异常提示的完整流程。对于微电子范围内的问题，模型通常能给出结构化解释，并可使用跨学科视角补充理解；对于范围外问题，Prompt 能在一定程度上约束模型拒答。

主要风险是模型仍可能出现细节错误，尤其在公式推导、工艺参数、教材定义差异较大的内容上，需要学生结合课程资料核对。

## ⑦ 应用价值与不足

### 应用价值

- **辅助课程学习：** 帮助学生快速理解 PN 结、MOSFET、CMOS 反相器等核心概念。
- **促进跨学科理解：** 通过物理、电路、材料、数学和工程视角解释问题，帮助学生建立知识联系。
- **降低使用门槛：** 静态网页部署，用户只需浏览器和自己的 DeepSeek API Key 即可使用。
- **提升复习效率：** 示例问题、对话记录和复制按钮方便学生整理笔记。
- **满足课程项目要求：** 使用真实公开 LLM，具有明确场景、完整流程、核心界面和基础测试。

### 不足

- **API Key 暴露风险：** 纯前端直连 API 会让用户填写的 Key 暴露在当前浏览器环境中，不适合正式生产系统。
- **缺少课程知识库：** 当前没有接入教材、课件或本地知识库，回答依赖模型自身知识。
- **没有微调训练：** 未进行 LoRA/QLoRA 微调，不能保证完全贴合某一门课程的大纲。
- **测试规模有限：** 当前测试主要是代表性人工测试，没有大规模自动化评测。
- **跨域与网络限制：** 浏览器直连 API 可能受到网络环境或 CORS 策略影响。

### 未来改进方向

- 增加后端代理，保护 API Key 并统一处理错误。
- 接入课程讲义、教材摘要或教师提供资料，构建 RAG 检索增强问答。
- 增加公式渲染能力，例如 KaTeX 或 MathJax。
- 增加历史对话保存、主题分类和学习卡片导出功能。
- 扩展自动化测试集，对准确性、稳定性和拒答能力进行量化评估。
